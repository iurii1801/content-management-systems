# Лабораторная работа №2. Введение в WordPress

## Цель работы

Научиться устанавливать WordPress в локальной среде, осваивать админ-панель, изменять внешний вид сайта через темы и расширять его функциональность с помощью плагинов.

## Ход работы

### Шаг 1. Подготовка среды

Для выполнения лабораторной работы необходимо подготовить локальную серверную среду, которая позволит запускать веб-приложения и работать с базой данных.

В качестве такой среды используется пакет **XAMPP**, включающий веб-сервер **Apache**, систему управления базами данных **MySQL (MariaDB)** и дополнительные инструменты для разработки.

#### Установка XAMPP

Сначала необходимо скачать установщик **XAMPP** с официального сайта:

```
https://www.apachefriends.org
```

После загрузки необходимо запустить установочный файл и выполнить стандартную процедуру установки.

В процессе установки выполняется распаковка файлов и настройка компонентов серверной среды.

![image](https://i.imgur.com/gCiOe8r.png)

#### Запуск серверной среды

После завершения установки необходимо открыть **XAMPP Control Panel** и запустить основные сервисы.

Для работы WordPress требуется запустить следующие модули:

* **Apache** — веб-сервер
* **MySQL** — сервер базы данных

После запуска модулей их статус должен отображаться как **Running**.

![image](https://i.imgur.com/xMoyGnw.png)

#### Проверка работы локального сервера

После запуска серверной среды необходимо убедиться, что веб-сервер работает корректно.

Для этого в браузере следует открыть адрес:

```
http://localhost
```

Если установка выполнена успешно, в браузере откроется стартовая страница **XAMPP Dashboard**.

![image](https://i.imgur.com/7Et1Ike.png)

#### Создание базы данных

Далее необходимо создать базу данных, которая будет использоваться системой WordPress.

Для этого следует перейти в инструмент управления базами данных **phpMyAdmin**.

Адрес для перехода:

```
http://localhost/phpmyadmin
```

В интерфейсе phpMyAdmin необходимо выполнить следующие действия:

1. Открыть вкладку **Databases (Базы данных)**.
2. В поле создания новой базы данных указать имя:

```
wp_lab2
```

3. Нажать кнопку **Create (Создать)**.

После выполнения этих действий база данных будет создана и готова для использования при установке WordPress.

![image](https://i.imgur.com/JrX8hL9.png)

### Шаг 2. Установка WordPress

После подготовки серверной среды и создания базы данных необходимо выполнить установку системы управления контентом **WordPress**.

#### Загрузка WordPress

Сначала необходимо скачать установочный архив WordPress с официального сайта проекта:

```
https://ru.wordpress.org/download/
```

![image](https://i.imgur.com/fqPcoZE.png)

После загрузки архив необходимо распаковать в директорию локального веб-сервера.

Для XAMPP используется папка:

```
xampp/htdocs/
```

В данной лабораторной работе была создана папка проекта:

```
wp_lab2
```

![image](https://i.imgur.com/r0QbFEO.png)

После распаковки файлов структура сайта становится доступной для запуска через браузер.

#### Запуск установщика WordPress

После копирования файлов необходимо открыть в браузере следующий адрес:

```
http://localhost/wp_lab2
```

После перехода по данному адресу запускается мастер установки WordPress.

На первом этапе отображается информационная страница, сообщающая о параметрах подключения к базе данных.

![image](https://i.imgur.com/ZrYwHgA.png)

После этого необходимо нажать кнопку **«Вперёд»**, чтобы перейти к настройке подключения к базе данных.

#### Настройка подключения к базе данных

На следующем этапе необходимо указать параметры базы данных, созданной ранее в **phpMyAdmin**.

В форме необходимо заполнить следующие поля:

| Параметр           | Значение                      |
| ------------------ | ----------------------------- |
| Имя базы данных    | `wp_lab2`                     |
| Имя пользователя   | `root`                        |
| Пароль             | (пустой или заданный в MySQL) |
| Сервер базы данных | `localhost`                   |
| Префикс таблиц     | `wp_`                         |

После заполнения параметров необходимо нажать кнопку **«Отправить»**.

![image](https://i.imgur.com/IFtYGOs.png)

После проверки подключения система предложит продолжить установку.

#### Настройка сайта

На следующем этапе необходимо указать основные параметры будущего сайта.

Необходимо заполнить следующие данные:

* название сайта;
* имя администратора;
* пароль администратора;
* адрес электронной почты.

Также можно указать параметр видимости сайта для поисковых систем.

![image](https://i.imgur.com/dBdAItc.png)

После заполнения всех данных необходимо нажать кнопку:

```
Установить WordPress
```

#### Завершение установки

После выполнения установки система сообщает об успешном завершении процесса.

На странице отображается информация о созданной учётной записи администратора.

![image](https://i.imgur.com/Iyqxumf.png)

После этого необходимо перейти на страницу входа в административную панель.

#### Вход в административную панель

Для входа в панель управления WordPress используется страница:

```
http://localhost/wp_lab2/wp-login.php
```

На странице входа необходимо указать:

* имя пользователя
* пароль администратора

![image](https://i.imgur.com/WDS8SlK.png)

После успешной авторизации открывается **административная панель WordPress**, через которую осуществляется управление сайтом.

### Шаг 4. Работа с темами

Темы в WordPress отвечают за внешний вид сайта: дизайн страниц, расположение элементов интерфейса и оформление контента.

Для управления темами необходимо открыть раздел:

```
Appearance (Внешний вид) → Themes (Темы)
```

#### Установка новой темы

Для установки новой темы необходимо нажать кнопку:

```

![image](https://i.imgur.com/8V240A8.png)

```

![image](https://i.imgur.com/ZVoBysY.png)

#### Изменение внешнего вида сайта

После активации новой темы внешний вид сайта автоматически изменяется.

На главной странице сайта появляется новый дизайн, изменяется структура элементов и оформление страниц.

### Шаг 5. Работа с плагинами

Плагины в WordPress позволяют расширять функциональность сайта без изменения исходного кода системы.
С их помощью можно добавлять новые возможности: формы обратной связи, редакторы, галереи, SEO-инструменты и многое другое.

Для управления плагинами используется раздел административной панели:

```
Plugins (Плагины) → Add New (Добавить плагин)
```

В данном разделе находится каталог доступных плагинов WordPress.

#### Установка плагина Classic Editor

В строке поиска необходимо ввести название плагина:

```
Classic Editor
```

После появления нужного плагина необходимо нажать кнопку:

```
Install Now (Установить сейчас)
```

После установки необходимо нажать кнопку:

```
Activate (Активировать)
```

![image](https://i.imgur.com/lMg671U.png)

Плагин **Classic Editor** возвращает классический редактор WordPress, который использовался в старых версиях системы.

После активации можно перейти в раздел:

```
Posts (Записи) → Add New (Добавить запись)
```

и убедиться, что используется **классический редактор записей**.

![image](https://i.imgur.com/iogr6tb.png)


#### Установка плагина Contact Form 7

Для добавления формы обратной связи необходимо установить плагин **Contact Form 7**.

В разделе установки плагинов в строке поиска необходимо ввести:

```

```

После нахождения плагина необходимо нажать:

```
Install Now (Установить сейчас)
```

После завершения установки необходимо нажать:

```

После активации в левом меню административной панели появляется новый раздел:

Для создания формы необходимо открыть раздел:

```

WordPress автоматически создаёт стандартный шаблон формы, содержащий следующие поля:

## Контрольные вопросы

### 1. Что делает тема в WordPress, а что — плагин?

Тема в WordPress отвечает за внешний вид сайта. Она определяет дизайн страниц, структуру расположения элементов, шрифты, цвета, оформление заголовков, меню и других визуальных компонентов интерфейса.

Плагин, в отличие от темы, добавляет новую функциональность сайту. Плагины расширяют возможности WordPress, например позволяют добавить формы обратной связи, SEO-инструменты, галереи, системы безопасности, интернет-магазин и другие функции.

Тема отвечает за **дизайн сайта**, а плагины — за **его функциональность**.

### 2. Почему при смене темы контент сайта не теряется?

Контент сайта (страницы, записи, изображения, комментарии) хранится в базе данных WordPress, а не в файлах темы.

Тема отвечает только за отображение этого контента на страницах сайта. Поэтому при смене темы изменяется только внешний вид сайта, а все данные остаются в базе данных и продолжают отображаться в новой теме.

### 3. Как можно изменить внешний вид сайта без редактирования кода?

В WordPress внешний вид сайта можно изменить через встроенный инструмент настройки темы.

Для этого используется раздел:

```
Appearance (Внешний вид) → Customize (Настроить)
```

В этом разделе можно изменить различные параметры сайта, например:

* логотип сайта
* название и описание
* цвета оформления
* фон сайта
* расположение элементов интерфейса

Все изменения отображаются в режиме предварительного просмотра и применяются после нажатия кнопки **Publish (Опубликовать)**.
