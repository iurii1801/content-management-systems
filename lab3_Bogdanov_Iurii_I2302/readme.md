# Лабораторная работа №3. Разработка простой темы WordPress

## Цель работы

Научиться создавать собственную тему WordPress, разобраться в её минимальной структуре и принципах работы шаблонов.

## Инструкция по запуску

1. Установить WordPress.
2. Скопировать папку темы в wp-content/themes.
3. Активировать тему в разделе Appearance → Themes(Внешний вид → Темы).

## Ход работы

### Шаг 1. Подготовка среды

Перед началом разработки пользовательской темы `WordPress` необходимо подготовить рабочую среду и убедиться, что система `WordPress` установлена и функционирует корректно.

В данной лабораторной работе используется локальная серверная среда **XAMPP**, включающая веб-сервер **Apache** и систему управления базами данных **MySQL (MariaDB)**. Ранее установленная система WordPress используется как основа для разработки пользовательской темы.

Все темы `WordPress` хранятся в специальной директории проекта:

```
wp-content/themes
```

В этой папке располагаются как стандартные темы WordPress, так и пользовательские темы, создаваемые разработчиком.

Для выполнения лабораторной работы в данной директории была создана новая папка темы:

```
usm-theme
```

В дальнейшем в этой папке будут размещаться все файлы пользовательской темы `WordPress`.

Также для удобства разработки был включён режим отладки `WordPress`.

Для этого был открыт файл конфигурации системы:

```

```

В данном файле был изменён параметр:

```php
define('WP_DEBUG', true);
```

Включение режима отладки позволяет отображать ошибки и предупреждения системы `WordPress`, что упрощает процесс разработки и тестирования темы.

### Шаг 2. Создание обязательных файлов темы

#### Структура папки темы

После создания директории пользовательской темы необходимо добавить в неё обязательные файлы, без которых `WordPress` не сможет распознать тему.

Минимальная тема WordPress должна содержать два файла:

* `style.css`
* `index.php`

Эти файлы являются обязательными для любой темы `WordPress`.

После создания файлов структура директории темы имеет следующий вид:

```
usm-theme
   style.css
   index.php
```

![image](https://i.imgur.com/EJJHMYn.png)

#### Создание файла `style.css`

В папке темы был создан файл:

```
style.css
```

В данный файл необходимо добавить **метаданные темы**, которые используются системой `WordPress` для отображения информации о теме в административной панели.

Пример метаданных темы:

```css
/*
Theme Name: USM Theme
Theme URI: http://localhost
Author: Student
Description: Простая тема WordPress для лабораторной работы
Version: 1.0
*/
```

После метаданных в файл необходимо добавить базовые CSS-правила для стилизации основных элементов сайта, таких как тело страницы, шапка и подвал.

![image](https://i.imgur.com/9LVpx1J.png)

#### Создание файла `index.php`

После этого был создан файл:

```
index.php
```

Данный файл является **главным шаблоном темы `WordPress`**.
Он используется системой для отображения содержимого сайта, если для конкретной страницы не существует специализированного шаблона.

В шаблоне подключаются основные части темы:

* `header.php` — шапка сайта;
* `footer.php` — подвал сайта.

В первой секции страницы выводится список последних записей сайта.
Для этого используется стандартный `WordPress Loop`, который позволяет получать записи из базы данных `WordPress` и отображать их на странице.

В шаблоне выводятся:

* заголовок записи;
* краткое описание записи;
* ссылка на страницу записи.

Также в файле реализованы дополнительные секции страницы, содержащие информационные блоки о возможностях темы и предоставляемых услугах. Эти блоки оформлены в виде карточек и отображаются с использованием сетки.

В конце страницы подключается файл `footer.php`, содержащий подвал сайта.

Пример содержимого файла:

```php
<?php get_header(); ?>

<section class="posts-section">
<h2 class="section-title">Последние записи</h2>

<div class="posts-container">

<?php
$count = 0;

if (have_posts()) :
while (have_posts()) : the_post();

if ($count == 5) break;
?>

<div class="post-card">
<h3><a href="<?php the_permalink(); ?>"><?php the_title(); ?></a></h3>
<p><?php the_excerpt(); ?></p>
</div>

<?php
$count++;
endwhile;
endif;
?>

</div>
</section>

<?php get_footer(); ?>
```

В данном коде:

* `get_header()` подключает файл `header.php`, содержащий шапку сайта;
* `<section class="posts-section">` создаёт секцию страницы для отображения последних записей;
* `<div class="posts-container">` формирует контейнер, в котором размещаются карточки записей;
* переменная `$count` используется для ограничения количества выводимых записей;
* `have_posts()` и `the_post()` реализуют стандартный `WordPress Loop`, который получает записи из базы данных;
* `if ($count == 5) break;` ограничивает вывод только пятью последними записями;
* `the_permalink()` выводит ссылку на страницу записи;
* `the_title()` выводит заголовок записи;
* `the_excerpt()` выводит краткое описание записи;
* `get_footer()` подключает файл `footer.php`, содержащий подвал сайта.

### Шаг 3. Общие части шаблонов

На данном этапе необходимо выполнить разделение шаблонов темы на общие части. Такой подход используется в `WordPress` для упрощения структуры темы и повторного использования кода на различных страницах сайта.

#### Создание файла `header.php`

В папке темы необходимо создать файл:

```
header.php
```

В данный файл необходимо перенести код шапки сайта, который располагается до начала основного содержимого страницы.

Файл должен содержать:

* объявление HTML-документа;
* блок `<head>`;
* вывод названия сайта;
* вывод описания сайта;
* открытие основного блока страницы.

Для корректной работы темы необходимо использовать функцию `WordPress`:

```php
<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title><?php bloginfo('name'); ?></title>
<?php wp_head(); ?>
</head>

<body>

<header>
<h1><?php bloginfo('name'); ?></h1>
<p><?php bloginfo('description'); ?></p>
</header>

<main>
```

Функция `wp_head()` позволяет `WordPress` подключать необходимые стили, скрипты и другие элементы.

![image](https://i.imgur.com/30Tp26J.png)

#### Создание файла `footer.php`

Далее необходимо создать файл:

```
footer.php
```

В данный файл необходимо перенести код подвала сайта, который располагается после основного содержимого страницы.

Файл содержит закрытие блока `<main>`, подвал сайта и функцию WordPress `wp_footer()`.

Пример содержимого файла:

```php
</main>

<footer>
<p>© <?php echo date('Y'); ?> My WordPress Theme</p>
</footer>

<?php wp_footer(); ?>

</body>
</html>
```

Функция `wp_footer()` используется системой WordPress для подключения скриптов и других элементов перед закрытием страницы.

![image](https://i.imgur.com/XwndftB.png)

#### Подключение шаблонов в `index.php`

После создания файлов `header.php` и `footer.php` необходимо подключить их в основном шаблоне `index.php`.

Для подключения шапки используется функция:

```php
<?php get_header(); ?>
```

Для подключения подвала используется функция:

```php
<?php get_footer(); ?>
```

Данные функции автоматически подключают соответствующие файлы шаблонов темы.

#### Вывод последних записей

На главной странице необходимо вывести список последних записей сайта. Для этого используется стандартный цикл WordPress.

В файле `index.php` необходимо реализовать следующий код:

```php
<?php get_header(); ?>

<h2>Последние записи</h2>

<?php
$count = 0;

if (have_posts()) :
    while (have_posts()) : the_post();

    if ($count == 5) break;
?>

<h3><a href="<?php the_permalink(); ?>"><?php the_title(); ?></a></h3>
<p><?php the_excerpt(); ?></p>

<?php
$count++;
    endwhile;
endif;
?>

<?php get_footer(); ?>
```

В процессе выполнения цикла выводятся:

* заголовок записи (`the_title()`);
* краткое описание записи (`the_excerpt()`).

С помощью переменной `$count` ограничивается количество выводимых записей, поэтому на главной странице отображаются **пять последних публикаций**.

### Шаг 4. Файл функций

На данном этапе необходимо создать файл функций темы `WordPress`.

Файл `functions.php` используется для добавления дополнительной функциональности теме. В данном файле можно подключать стили и скрипты, регистрировать меню, виджеты и добавлять различные возможности темы.

#### Создание файла `functions.php`

В папке темы необходимо создать файл:

```
functions.php
```

Данный файл автоматически загружается системой `WordPress` при активации темы и позволяет добавлять пользовательские функции.

#### Подключение стилей темы

Для корректного подключения стилей темы необходимо использовать функцию `WordPress` `wp_enqueue_style()`.

В файл `functions.php` необходимо добавить следующий код:

```php
<?php

function usm_theme_styles() {

    wp_enqueue_style(
        'usm-style',
        get_stylesheet_uri()
    );

}

add_action('wp_enqueue_scripts', 'usm_theme_styles');

?>
```

![image](https://i.imgur.com/SNFj0RX.png)

Функция `wp_enqueue_style()` используется для подключения CSS-файлов темы.

Функция `get_stylesheet_uri()` возвращает путь к файлу `style.css`, который находится в корневой папке темы.

Для выполнения данной функции используется хук `WordPress`:

```php
add_action('wp_enqueue_scripts', 'usm_theme_styles');
```

Данный хук сообщает системе `WordPress` о необходимости выполнить функцию подключения стилей при загрузке страницы.

#### Проверка структуры темы

После создания файла `functions.php` структура директории темы имеет следующий вид:

```
usm-theme
   style.css
   index.php
   header.php
   footer.php
   functions.php
```

### Шаг 5. Дополнительные шаблоны

На данном этапе необходимо создать дополнительные шаблоны темы `WordPress`, которые отвечают за отображение отдельных элементов сайта: записей, страниц, архивов и комментариев.

К таким шаблонам относятся:

* `single.php` — шаблон отдельной записи
* `page.php` — шаблон страницы
* `sidebar.php` — шаблон боковой панели
* `comments.php` — шаблон комментариев
* `archive.php` — шаблон архива записей

#### Создание шаблона записи

Для отображения отдельной записи необходимо создать файл **`single.php`**.

Данный файл используется `WordPress` для вывода страницы записи блога.
В шаблоне выводится заголовок записи, её содержимое, комментарии, а также подключаются основные элементы темы: шапка сайта, боковая панель и подвал.

Пример содержимого файла:

```php
<?php get_header(); ?>

<section class="content-page">
    <div class="content-main">
        <h2 class="section-title">Запись</h2>

        <?php
        if (have_posts()) :
            while (have_posts()) : the_post();
        ?>

        <article class="content-card">
            <h3><?php the_title(); ?></h3>
            <div class="content-text">
                <?php the_content(); ?>
            </div>
        </article>

        <?php comments_template(); ?>

        <?php
            endwhile;
        endif;
        ?>
    </div>

    <div class="content-side">
        <?php get_sidebar(); ?>
    </div>
</section>

<?php get_footer(); ?>
```

В данном коде:

* `get_header()` подключает файл `header.php`, содержащий шапку сайта;
* `<section class="content-page">` создаёт основной контейнер страницы записи;
* `<div class="content-main">` формирует область для отображения основного содержимого записи;
* `have_posts()` и `the_post()` реализуют стандартный `WordPress Loop`, который проверяет наличие записей и подготавливает данные записи для вывода;
* `the_title()` выводит заголовок записи;
* `the_content()` выводит полный текст записи, включая изображения и форматирование;
* `comments_template()` подключает шаблон комментариев `comments.php`, который отображает список комментариев и форму их добавления;
* `<div class="content-side">` создаёт область для боковой панели;
* `get_sidebar()` подключает файл `sidebar.php`, содержащий навигацию или дополнительные элементы сайта;
* `get_footer()` подключает файл `footer.php`, который содержит подвал сайта.

#### Создание шаблона страницы

Для отображения статических страниц сайта необходимо создать файл **`page.php`**.

Этот шаблон используется `WordPress` для вывода страниц сайта, таких как **Главная**, **Контакты**, **О нас** и других.

Пример содержимого файла:

```php
<?php get_header(); ?>

<section class="content-page">
    <div class="content-main">
        <h2 class="section-title">Страница</h2>

        <?php
        if (have_posts()) :
            while (have_posts()) : the_post();
        ?>

        <article class="content-card">
            <h3><?php the_title(); ?></h3>
            <div class="content-text">
                <?php the_content(); ?>
            </div>
        </article>

        <?php comments_template(); ?>

        <?php
            endwhile;
        endif;
        ?>
    </div>

    <div class="content-side">
        <?php get_sidebar(); ?>
    </div>
</section>

<?php get_footer(); ?>
```

В данном коде:

* `get_header()` подключает файл `header.php`, содержащий шапку сайта;
* `<section class="content-page">` создаёт основной контейнер страницы;
* `<div class="content-main">` формирует область для отображения основного содержимого страницы;
* `have_posts()` и `the_post()` реализуют стандартный `WordPress Loop`, который проверяет наличие страниц и подготавливает данные страницы для вывода;
* `the_title()` выводит заголовок страницы;
* `the_content()` выводит содержимое страницы, включая текст, изображения и другое форматирование;
* `comments_template()` подключает шаблон комментариев `comments.php`, позволяющий отображать комментарии и форму их добавления;
* `<div class="content-side">` создаёт область для боковой панели;
* `get_sidebar()` подключает файл `sidebar.php`, содержащий элементы навигации или дополнительную информацию;
* `get_footer()` подключает файл `footer.php`, содержащий подвал сайта.

#### Создание боковой панели

Для отображения боковой панели сайта необходимо создать файл **`sidebar.php`**.

В данном файле размещается блок навигации сайта.

Пример содержимого файла:

```php
<aside class="sidebar-box">
    <h3>Навигация</h3>
    <ul>
        <li><a href="<?php echo home_url(); ?>">Главная страница</a></li>
        <li><a href="<?php echo home_url('/?page_id=2'); ?>">Страницы сайта</a></li>
        <li><a href="<?php echo home_url(); ?>">Записи блога</a></li>
    </ul>
</aside>
```

В данном коде:

* `<aside class="sidebar-box">` создаёт контейнер боковой панели сайта;
* `<h3>` выводит заголовок блока навигации;
* `<ul>` формирует список элементов навигации;
* `<li>` создаёт отдельные пункты меню;
* `<a>` создаёт ссылки для перехода по страницам сайта;
* `home_url()` возвращает базовый URL сайта `WordPress`;
* `home_url('/?page_id=2')` формирует ссылку на конкретную страницу сайта по её идентификатору.

Данный файл используется для отображения боковой панели сайта и подключается в шаблонах с помощью функции:

```php
<?php get_sidebar(); ?>
```

#### Создание шаблона комментариев

Для отображения комментариев необходимо создать файл **`comments.php`**.

Данный файл отвечает за вывод списка комментариев и формы их добавления.

Пример содержимого файла:

```php
<?php
if (have_comments()) :
?>

    <h3>Комментарии</h3>

    <?php
    wp_list_comments();
endif;

if (comments_open()) :
    comment_form();
endif;
?>
```

В данном коде:

* `have_comments()` проверяет, существуют ли комментарии к текущей записи или странице;
* `<h3>` выводит заголовок блока комментариев;
* `wp_list_comments()` выводит список всех комментариев, относящихся к записи;
* `comments_open()` проверяет, разрешено ли добавление новых комментариев;
* `comment_form()` выводит форму, позволяющую пользователям оставить новый комментарий.

Шаблон комментариев подключается в файле **`single.php`** с помощью функции:

```php
<?php comments_template(); ?>
```

#### Создание шаблона архива

Для отображения архивов записей необходимо создать файл **`archive.php`**.

Данный шаблон используется `WordPress` для вывода списка записей по рубрикам, дате публикации и другим параметрам.

Пример содержимого файла:

```php
<?php get_header(); ?>

<h2>Архив записей</h2>

<?php
if (have_posts()) :
    while (have_posts()) : the_post();
?>

    <h3><?php the_title(); ?></h3>
    <p><?php the_excerpt(); ?></p>

<?php
    endwhile;
endif;
?>

<?php get_sidebar(); ?>

<?php get_footer(); ?>
```

В данном коде:

* `get_header()` подключает файл `header.php`, содержащий шапку сайта;
* `<h2>` выводит заголовок страницы архива записей;
* `have_posts()` и `the_post()` реализуют стандартный `WordPress Loop`, который проверяет наличие записей и подготавливает их для вывода;
* `the_title()` выводит заголовок каждой записи;
* `the_excerpt()` выводит краткое описание записи (анонс);
* `get_sidebar()` подключает файл `sidebar.php`, содержащий боковую панель сайта;
* `get_footer()` подключает файл `footer.php`, содержащий подвал сайта.

## Контрольные вопросы

### 1. Какие два файла являются обязательными для любой темы WordPress?

Для того чтобы `WordPress` распознал пользовательскую тему, она должна содержать как минимум два обязательных файла.

Первый файл — **style.css**.
В этом файле содержатся метаданные темы (название темы, автор, описание, версия). `WordPress` считывает эту информацию и отображает тему в административной панели.

Второй файл — **index.php**.
Этот файл является основным шаблоном темы. Он используется системой для вывода содержимого сайта, если для страницы не существует более специализированного шаблона.

### 2. Как подключаются общие части шаблонов (`header`, `footer`, `sidebar`)?

В WordPress общие части шаблонов (шапка сайта, подвал и боковая панель) подключаются с помощью специальных функций.

Для подключения шапки сайта используется функция:

```
get_header();
```

Для подключения подвала сайта используется функция:

```
get_footer();
```

Для подключения боковой панели используется функция:

```
get_sidebar();
```

Эти функции позволяют разделить структуру сайта на отдельные файлы шаблонов и использовать их повторно в разных страницах темы.

Например, файл **`header.php`** подключается в шаблонах страниц, чтобы отобразить верхнюю часть сайта, а файл **`footer.php`** — для отображения нижней части сайта.

### 3. Чем отличаются `index.php`, `single.php` и `page.php`?

В WordPress разные типы страниц могут использовать разные шаблоны.

Файл **`index.php`** является главным шаблоном темы. Он используется системой для отображения страниц сайта, если для них не существует более специализированных шаблонов.

Файл **`single.php`** используется для отображения отдельной записи блога. Когда пользователь открывает конкретную запись, WordPress использует именно этот шаблон.

Файл **`page.php`** используется для отображения статических страниц сайта, например страниц «О нас», «Контакты» или других информационных страниц.

### 4. Зачем нужен файл `functions.php` в теме?

Файл **`functions.php`** используется для добавления функциональности теме WordPress.

В этом файле можно подключать стили и скрипты, регистрировать меню сайта, добавлять поддержку различных возможностей WordPress и расширять функциональность темы.

Например, в файле **`functions.php`** можно подключить файл стилей темы с помощью функции:

```
wp_enqueue_style()
```

Также в этом файле можно регистрировать области виджетов, добавлять поддержку миниатюр записей, настраивать меню навигации и выполнять другие действия, связанные с работой темы.

Файл **`functions.php`** позволяет расширять возможности темы и управлять её функциональностью.
